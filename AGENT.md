# Agent 지침

## 언어 정책

**중요: 모든 대화는 한국어로만 진행합니다.**

- 모든 응답은 한국어로 작성해야 합니다
- 코드 주석도 한국어로 작성합니다
- 커밋 메시지도 한국어로 작성합니다
- 문서화도 한국어로 작성합니다
- 기술 용어는 필요시 영어를 병기할 수 있습니다 (예: "컴포넌트(component)")

## 예외 사항

다음의 경우에만 영어 사용이 허용됩니다:
- 코드 자체 (변수명, 함수명, 클래스명 등)
- 라이브러리/프레임워크 이름
- 파일명
- Git 명령어
- 기술 문서의 원문 인용

## 작업 관리 및 문서화

**작업 중 중요한 내용은 자동으로 문서에 반영합니다.**

- 사용자와 진행한 작업 중 중요한 결정사항, 구조 변경, 설정 등은 알아서 관련 문서를 업데이트합니다
- 별도 요청 없이도 프로젝트의 일관성 유지를 위해 필요한 문서화를 수행합니다
- 중요한 변경사항이나 의사결정은 적절한 문서(README, 기술 문서 등)에 기록합니다

## Git 작업 정책

**Git 관련 작업은 반드시 GitHub MCP를 사용합니다.**

- `git push`, `git commit` 등의 git 명령어 대신 MCP 도구를 사용합니다
- GitHub MCP 도구: `mcp__plugin_github_github__push_files` 사용
- 인증 문제를 피하고 안정적인 작업을 위해 MCP를 우선 사용합니다
- git 명령어 직접 사용은 인증 설정이 완료된 경우에만 허용됩니다

## 브랜치 관리 정책

**모든 수정 및 개발은 새로운 브랜치를 만들어서 진행합니다.**

- main 브랜치에서 직접 작업하지 않습니다
- 새로운 기능, 버그 수정, 리팩토링 등 모든 작업은 별도의 브랜치에서 진행합니다
- 브랜치 이름은 작업 내용을 명확하게 표현해야 합니다 (예: `feature/gallery-lightbox`, `fix/mobile-layout`, `refactor/css-cleanup`)
- 작업 완료 후 main 브랜치로 병합합니다
- 병합 후에는 작업이 완료된 브랜치를 정리할 수 있습니다

## 개인정보 보호

**민감한 정보는 절대 커밋하거나 업로드하지 않습니다.**

- API 토큰, 액세스 키, 시크릿 키 등의 인증 정보
- 계정 정보 (아이디, 패스워드, 이메일 등)
- IP 주소, 호스트명 등의 네트워크 정보
- 개인 식별 정보 (주민등록번호, 전화번호, 주소 등)
- .env 파일, credentials 파일 등 민감 정보를 포함한 설정 파일
- 이러한 정보가 포함된 파일을 발견하면 사용자에게 경고하고 .gitignore에 추가합니다
